# AI Personal Trainer in Claude Cowork
## Building a Clickable Mobile App Prototype

### Overview

## Setup Instructions

### Step 1: Open Claude Cowork
**⏱️ Time: 1 minute**

Go to Claude Cowork on your desktop or web app. You're ready to generate a prototype. No setup needed.

### Step 2: Generate a Clickable Mobile App Prototype
**⏱️ Time: 2-3 minutes**

Copy and paste the prompt below directly into Claude Cowork. This prompt creates a fully interactive, clickable mobile app prototype that you can click through in your browser.

**PROMPT TO COPY:**

Generate a clickable prototype of a mobile app for TrainAI, an AI personal trainer. Create a single interactive HTML page with three functional screens: (1) Home Screen showing vital signs dashboard with sleep, steps, workouts, and recovery stats; (2) Vitals Update Screen with clickable sliders and dropdowns so users can input their sleep, energy, nutrition, and pain levels; (3) Training Plan Screen that shows how their vitals triggered real-time adjustments to their workout (e.g., "4 hours sleep → reduce volume by 25% and extend rest"). The app must be fully clickable with working buttons, functional form inputs, smooth screen transitions, and mobile-responsive design. Use modern, minimalist styling with light grays and blues. This is a prototype to demonstrate how AI fitness coaching adapts to real user data instantly.

### Step 3: Set Model to Haiku 4.5 and Generate
**⏱️ Time: 30 seconds (Haiku generates prototype in 15-30 seconds)**

1. Before running the prompt, select: Claude Model = Haiku 4.5 (Cowork settings)
2. Paste the prompt from Step 2 into the chat
3. Hit Enter—Haiku generates your fully interactive prototype in seconds
4. Click the preview to interact with your clickable prototype immediately

### Step 4: Customize Your Prototype
**⏱️ Time: 5-10 minutes per customization (each iteration regenerates in 15-30 seconds)**

1. Add custom user data: "Customize this for Sarah, 32, recovering from ACL surgery with knee pain"
2. Request design changes: "Add dark mode", "Change colors to orange and teal"
3. Add vitals: "Include HRV data and menstrual cycle tracking"
4. Extend functionality: "Add workout history chart", "Include nutrition logging"
5. All changes regenerate instantly with Haiku 4.5

## Example: Create a Custom Prototype

After Step 3, ask Haiku:

"Customize this prototype for Marcus, 38, a software engineer with:
- Goals: Build strength for climbing
- Injuries: Lower back pain (2 years)
- Sleep: Usually 6-7 hours (but often only 5)
- Equipment: Home gym with dumbbells, pull-up bar, resistance bands
- Pain level: Fluctuates 2-5 out of 10 depending on sleep

When Marcus has less than 6 hours sleep, the app should automatically reduce intensity by 15% and add mobility work."

Haiku regenerates a fully clickable prototype for Marcus in seconds.

## Best Practices

- Test the prototype: Click every button, update every vital, verify plans adjust realistically
- Keep prompts specific: Include user age, injury history, and goals for better personalization
- Iterate quickly: Ask for design changes, new features, or customizations—Haiku re-generates in seconds
- Use for training: Show your team this prototype to demonstrate AI fitness coaching in action
- Collect feedback: "What features would make this more useful?" helps refine the next iteration
- Export the code: Once satisfied, download the HTML and integrate into your platform
- **Optional—Schedule recurring coaching:** Use Claude Cowork scheduled tasks to run this prompt daily for a specific user, generating fresh plans each morning based on their sleep/vitals
